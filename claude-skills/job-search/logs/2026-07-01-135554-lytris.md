## Job Description

About Lytris
Lytris is an AI platform for STEM, using university math as its wedge: deep integration with math faculties cuts grading and prep
time and delivers real-time feedback to students. Backed by a €250k pre-seed, we are in contract negotiations with our first
university and piloting with half of Germany's Universities of Excellence. We ship fast as a lean, remote-first team.
The Role
Own user-facing features end to end, from the data model to the React UI. You will build the agentic LLM features that grade
university math and give students real-time feedback, and ship them to the faculties piloting Lytris today.
What You'll Do
• Build and own AI features that grade math and give students feedback: agentic, tool-calling flows, not one-off prompts.
• Design real-time data models and keep them correct and fast under real usage.
• Integrate auth, billing, and secure file storage cleanly across the stack.
• Decide what to build, not just how: talk to faculty at our pilot universities and turn what you see into the next feature.
• Write tests for what you ship and validate it against real usage.
Must-Have Skills
• 1–3 years building full-stack TypeScript apps with React and Next.js (App Router), comfortable from UI to backend.
• Worked with a real-time/reactive backend (Convex, Firebase, Supabase, or similar): typed queries, mutations, schema.
• Built LLM features in a real project (RAG, tool-calling, or agent flows) and keen to go deeper into agentic engineering.
• Comfortable with Tailwind and a component system (shadcn/ui, Radix, or similar); can build clean, accessible UI.
• Startup autonomy: handles ambiguity, ships across the stack, and tests/typechecks before pushing.

## Answers

**Pitch:**

The core of what Lytris needs — agentic LLM features grounded in real data, shipped to real users — is exactly what I built at Credible Informatics. I built an AI research dashboard where retail investors could ask free-form questions about any company and get GPT answers grounded in real earnings transcripts, financial news, and YouTube commentary. The data layer was Scrapy crawlers ingesting 10+ financial sources on a cron schedule, OpenAI embeddings stored in a vector DB, and Redis caching to keep latency under 200ms. That is a RAG pipeline with agentic ambitions, not a one-off prompt. It shipped to real users and held up.

I work in TypeScript, Next.js, and Python. I have shipped full-stack features including auth (JWT and Google OAuth, deployed to 1,000+ users) and deployed stacks end to end via GitHub Actions. The real-time backend gap — Convex, Firebase, Supabase — is honest. My reactive data experience is PostgreSQL and Redis, not a typed live-query system. That said, the conceptual model is the same: schema ownership, mutation correctness, keeping reads fast under load. I'd pick up Convex fast.

What pulls me to Lytris specifically is the feedback loop. Grading math is a hard agentic problem — the model has to reason about correctness, not just pattern-match. And the pilot structure (real faculties, real students, real friction) means the feedback is immediate and specific. That is the best environment for building AI features that actually work.
